\chapter{System Design and Methodology}

\section{System Overview}

ShadeAI is organised as a six-stage, pipeline-centric architecture. Each stage operates on a well-defined input–output contract, so any individual module can be tested, swapped, or upgraded without rippling through the rest of the system. Five pillars anchor the design:

\begin{itemize}
    \item \textbf{Modularity:} Every component is self-contained — its tests, configuration, and failure modes are isolated from its neighbours
    \item \textbf{Scalability:} The pipeline handles both single-image interactive queries and multi-hundred-image batch jobs without architectural changes
    \item \textbf{Accessibility:} Zero training data, zero GPU dependency
    \item \textbf{Accuracy:} Classical computer-vision and colour-science techniques, carefully composed, deliver research-grade $\Delta E$
    \item \textbf{Usability:} Three interface layers (CLI, Web, REST API) serve distinct user profiles
\end{itemize}

\section{Pipeline Modules}

\subsection{Module 1: Image Preprocessing}

\subsubsection{Purpose}
The preprocessor normalises colour representation before any downstream analysis. It corrects illuminant-induced casts, converts into the colour spaces subsequent modules require, and runs a first-pass quality sanity check.

\subsubsection{White Balance Correction}

White-balance correction neutralises the colour cast imposed by the scene illuminant. ShadeAI uses the Gray World algorithm — a computationally lightweight method that assumes the spatial average of the image should be achromatic:

\textbf{Algorithm:}
\begin{enumerate}
    \item Calculate average RGB values: $\bar{R}, \bar{G}, \bar{B}$
    \item Compute gray value: $gray = (\bar{R} + \bar{G} + \bar{B}) / 3$
    \item Calculate scaling factors:
    \begin{align}
        k_R &= gray / \bar{R} \\
        k_G &= gray / \bar{G} \\
        k_B &= gray / \bar{B}
    \end{align}
    \item Apply corrections: $R' = R \times k_R$, $G' = G \times k_G$, $B' = B \times k_B$
    \item Clip values to [0, 255]
\end{enumerate}

\textbf{Rationale:} Gray World requires no calibration target and executes in $O(n)$ time. For the selfie-style images ShadeAI targets — typically captured under indoor lighting or diffuse daylight — this normalisation removes the majority of colour-cast artefacts.

\subsubsection{Color Space Conversions}

Different pipeline stages consume different colour representations:

\begin{itemize}
    \item \textbf{RGB:} Input format, used for display
    \item \textbf{CIELAB:} Used for ITA° calculation and color matching
    \item \textbf{HSV:} Used for skin segmentation
    \item \textbf{YCbCr:} Used for skin segmentation
\end{itemize}

\textbf{RGB to CIELAB Conversion:}
\begin{enumerate}
    \item Convert RGB to XYZ using standard transformation matrix
    \item Normalize XYZ by reference white point (D65)
    \item Apply nonlinear transformation:
    \begin{equation}
        f(t) = \begin{cases}
            t^{1/3} & \text{if } t > (6/29)^3 \\
            \frac{1}{3}(29/6)^2 t + 4/29 & \text{otherwise}
        \end{cases}
    \end{equation}
    \item Calculate LAB values:
    \begin{align}
        L^* &= 116 \cdot f(Y/Y_n) - 16 \\
        a^* &= 500 \cdot (f(X/X_n) - f(Y/Y_n)) \\
        b^* &= 200 \cdot (f(Y/Y_n) - f(Z/Z_n))
    \end{align}
\end{enumerate}

\subsection{Module 2: Face Detection}

\subsubsection{Purpose}
Without isolating the face, background and clothing pixels would contaminate the CIELAB centroid. This module draws a bounding box around the primary face and extracts a padded crop for downstream analysis.

\subsubsection{Haar Cascade Classifier}

ShadeAI uses OpenCV's pre-trained Haar Cascade classifier for face detection.

\textbf{Algorithm:}
\begin{enumerate}
    \item Convert image to grayscale
    \item Apply Haar Cascade classifier with parameters:
    \begin{itemize}
        \item scaleFactor = 1.1 (image pyramid scale)
        \item minNeighbors = 5 (minimum detections for confirmation)
        \item minSize = (30, 30) (minimum face size)
    \end{itemize}
    \item Select largest detected face (if multiple)
    \item Extract face region with 10\% padding
    \item Validate detection (minimum size, aspect ratio)
\end{enumerate}

\textbf{Advantages:}
\begin{itemize}
    \item Sub-millisecond inference on a modern CPU core
    \item Ships pre-trained inside OpenCV — no fine-tuning step
    \item Tolerates moderate variation in pose and illumination
    \item Deterministic, reproducible output
\end{itemize}

\textbf{Limitations:}
\begin{itemize}
    \item Recall drops steeply beyond roughly 30° off-axis
    \item Occlusions (sunglasses, masks) often cause missed detections
    \item Precision trails deep-learning detectors on cluttered backgrounds
\end{itemize}

\subsubsection{Fallback Mechanisms}

If the primary pass yields no detection the module escalates through two fallback tiers:
\begin{enumerate}
    \item Retry with relaxed parameters (minNeighbors = 3)
    \item Analyse the entire frame and flag reduced confidence to the user
    \item Surface a quality warning recommending a better-lit, frontal retake
\end{enumerate}

\subsection{Module 3: Skin Segmentation}

\subsubsection{Purpose}
Within the face crop, eyes, brows, lips, and any residual background must be excluded. Only verified skin pixels feed into the feature-extraction stage.

\subsubsection{Rule-Based Segmentation}

ShadeAI fuses thresholds from two complementary colour spaces:

\textbf{HSV Thresholds:}
\begin{align}
    0° &\leq H \leq 50° \\
    0.2 &\leq S \leq 0.8 \\
    0.3 &\leq V \leq 1.0
\end{align}

\textbf{YCbCr Thresholds:}
\begin{align}
    77 &\leq Cb \leq 127 \\
    133 &\leq Cr \leq 173
\end{align}

\textbf{Algorithm:}
\begin{enumerate}
    \item Convert face region to HSV and YCbCr
    \item Apply thresholds to create binary masks
    \item Combine masks using logical AND
    \item Apply morphological operations:
    \begin{itemize}
        \item Opening (erosion + dilation) to remove noise
        \item Dilation to fill small gaps
    \end{itemize}
    \item Extract skin pixels using mask
    \item Validate (minimum 1000 pixels)
\end{enumerate}

\textbf{Rationale:} Neither HSV nor YCbCr alone captures skin reliably across the full MST range. HSV excels at hue discrimination but falters under saturated artificial light; YCbCr is more luminance-invariant but occasionally admits non-skin reds. Their intersection prunes most false positives while retaining genuine skin across tones from MST-01 to MST-10.

\subsubsection{Morphological Operations}

Post-threshold clean-up relies on standard morphological filters:

\begin{itemize}
    \item \textbf{Opening:} Erodes then dilates — scrubs isolated noise islands
    \item \textbf{Dilation:} Expands surviving regions to reconnect skin patches broken by eyelashes or fine wrinkles
    \item \textbf{Kernel Size:} 5$\times$5 pixels, chosen empirically as the best trade-off between noise suppression and boundary fidelity
\end{itemize}

\subsection{Module 4: Feature Extraction}

\subsubsection{Purpose}
This module distils the high-dimensional pixel data into a compact descriptor — the numbers that classification and matching will actually operate on.

\subsubsection{ITA° Calculation}

ITA° condenses skin lightness into a single angular scalar derived from CIELAB coordinates:

\begin{equation}
ITA° = \arctan\left(\frac{L^* - 50}{b^*}\right) \times \frac{180}{\pi}
\end{equation}

where:
\begin{itemize}
    \item $L^*$ is the mean lightness of skin pixels
    \item $b^*$ is the mean yellow-blue component
\end{itemize}

\textbf{Interpretation:}
\begin{itemize}
    \item Higher ITA° → Lighter skin tone
    \item Lower ITA° → Darker skin tone
    \item ITA° > 55° → Very fair (MST-01)
    \item ITA° < -30° → Deep (MST-10)
\end{itemize}

\subsubsection{CIELAB Statistics}

Beyond the scalar ITA°, the module computes first-order statistics across all three CIELAB channels:

\begin{align}
    \bar{L^*} &= \frac{1}{N} \sum_{i=1}^{N} L_i^* \\
    \bar{a^*} &= \frac{1}{N} \sum_{i=1}^{N} a_i^* \\
    \bar{b^*} &= \frac{1}{N} \sum_{i=1}^{N} b_i^*
\end{align}

where $N$ is the number of skin pixels.

These means serve three distinct downstream consumers:
\begin{itemize}
    \item ITA° computation (tone depth)
    \item Undertone inference ($a^*/b^*$ ratios)
    \item CIEDE2000 shade matching (direct CIELAB-to-CIELAB comparison)
\end{itemize}

\subsubsection{MST Classification}

\textbf{Algorithm:}
\begin{verbatim}

    class = MST-01

    class = MST-03
...

    class = MST-10
\end{verbatim}

\subsubsection{Undertone Detection}

Undertone classification uses a weighted heuristic that accumulates evidence from $a^*$, $b^*$, and ITA°:

\textbf{Algorithm:}
\begin{enumerate}
    \item Initialize score = 0
    \item If $b^* > 15$: score += 0.4 (yellow → warm)
    \item If $b^* < 10$: score -= 0.4 (blue → cool)
    \item If $a^* > 12$: score += 0.3 (red → warm)
    \item If $a^* < 8$: score -= 0.3 (pink → cool)
    \item If ITA° > 28: score += 0.2 (lighter → warm)
    \item If ITA° < 10: score -= 0.2 (darker → cool)
    \item Clip score to [-1.0, 1.0]
    \item Classify:
    \begin{itemize}
        \item score > 0.3 → Warm
        \item score < -0.3 → Cool
        \item otherwise → Neutral
    \end{itemize}
\end{enumerate}

\textbf{Rationale:} The thresholds were tuned against ground-truth images whose undertones were independently assessed by trained makeup artists. Warm skin tends to exhibit elevated $b^*$ (yellow) and $a^*$ (red) channels; cool skin suppresses both. The ITA° term adds a weak lightness prior observed empirically.

\subsection{Module 6: Shade Recommendation}

\subsubsection{Purpose}
This final stage ranks every shade in the database by its perceptual proximity to the user's extracted skin colour, then returns the best five matches.

\subsubsection{CIEDE2000 Implementation}

ShadeAI implements the full CIE-specified $\Delta E_{00}$ formula, including the rotation term and neutral-colour compensation that earlier $\Delta E$ variants omit:

\textbf{Algorithm Steps:}
\begin{enumerate}
    \item Calculate chroma: $C = \sqrt{a^{*2} + b^{*2}}$
    \item Compute $\bar{C}$ and apply G correction to $a^*$
    \item Calculate hue angle: $h' = \arctan(b^*/a'^*)$
    \item Compute differences: $\Delta L'$, $\Delta C'$, $\Delta H'$
    \item Calculate weighting functions: $S_L$, $S_C$, $S_H$
    \item Compute rotation term: $R_T$
    \item Calculate final $\Delta E$:
    \begin{equation}
        \Delta E_{00} = \sqrt{\left(\frac{\Delta L'}{S_L}\right)^2 + \left(\frac{\Delta C'}{S_C}\right)^2 + \left(\frac{\Delta H'}{S_H}\right)^2 + R_T \frac{\Delta C'}{S_C} \frac{\Delta H'}{S_H}}
    \end{equation}
\end{enumerate}

\textbf{Interpretation:}
\begin{itemize}
    \item $\Delta E$ < 1.0: Imperceptible difference (perfect match)
    \item $\Delta E$ < 2.0: Outstanding match
    \item $\Delta E$ < 3.0: Excellent match (industry standard)
    \item $\Delta E$ < 5.0: Good match (acceptable)
    \item $\Delta E$ $\geq$ 5.0: Poor match (noticeable difference)
\end{itemize}

\subsubsection{Recommendation Algorithm}

\textbf{Algorithm:}
\begin{enumerate}
    \item Load foundation database (187 shades)
    \item For each shade:
    \begin{itemize}
        \item Calculate $\Delta E$ using CIEDE2000
        \item Check undertone compatibility
        \item Store result with metadata
    \end{itemize}
    \item Sort results:
    \begin{itemize}
        \item Primary: Undertone-compatible shades by $\Delta E$
        \item Secondary: Incompatible shades by $\Delta E$
    \end{itemize}
    \item Return top 5 recommendations
    \item Include for each:
    \begin{itemize}
        \item Brand and shade name
        \item $\Delta E$ value
        \item CIELAB values
        \item Undertone
        \item MST range
        \item Match quality indicator
    \end{itemize}
\end{enumerate}

\textbf{Undertone Compatibility:}
\begin{itemize}
    \item Neutral undertone: Compatible with all shades
    \item Warm/Cool: Prefer matching undertone
    \item If insufficient matches: Include best non-matching
\end{itemize}

\section{Photo Quality Assessment}

\subsection{Purpose}
Input quality is the single largest source of matching error. Rather than letting a poor image silently degrade results, ShadeAI evaluates six metrics up front and provides corrective guidance before analysis begins.

\subsection{Quality Metrics}

Six quality metrics are evaluated:

\begin{enumerate}
    \item \textbf{Brightness:}
    \begin{itemize}
        \item Measure: Mean pixel intensity
        \item Optimal: 100-180
        \item Issues: Too dark (< 80) or too bright (> 200)
    \end{itemize}
    
    \item \textbf{Blur:}
    \begin{itemize}
        \item Measure: Laplacian variance
        \item Optimal: > 100
        \item Issues: Blurry image (< 100)
    \end{itemize}
    
    \item \textbf{Color Cast:}
    \begin{itemize}
        \item Measure: RGB channel differences
        \item Optimal: Balanced channels
        \item Issues: Strong color cast (difference > 30)
    \end{itemize}
    
    \item \textbf{Face Detection:}
    \begin{itemize}
        \item Measure: Face detection success
        \item Optimal: Face detected with confidence > 0.8
        \item Issues: No face or low confidence
    \end{itemize}
    
    \item \textbf{Resolution:}
    \begin{itemize}
        \item Measure: Image dimensions
        \item Optimal: > 640$\times$480
        \item Issues: Low resolution
    \end{itemize}
    
    \item \textbf{Exposure:}
    \begin{itemize}
        \item Measure: Histogram distribution
        \item Optimal: Well-distributed
        \item Issues: Clipping or narrow range
    \end{itemize}
\end{enumerate}

\subsection{Quality Scoring}

A composite score maps the six metrics onto a 0–100 scale:
\begin{equation}
Score = 100 - \sum_{i=1}^{6} penalty_i
\end{equation}

Each metric contributes a penalty between 0 and 25, scaled by severity.

\textbf{Quality Levels:}
\begin{itemize}
    \item Excellent: Score $\geq$ 90
    \item Good: Score $\geq$ 75
    \item Fair: Score $\geq$ 60
    \item Poor: Score < 60
\end{itemize}

\section{User Interfaces}

\section{Summary}

This chapter has laid out ShadeAI's architecture and algorithms in full: six pipeline stages, a three-tier deployment structure, a 187-shade database with deliberate deep-tone emphasis, and three interface surfaces targeting different user profiles. Two themes run through every design choice — keep the inference path free of GPU and training dependencies, and never sacrifice darker-tone accuracy for lighter-tone convenience. The next chapter presents the concrete implementation, experimental evaluation, and head-to-head performance comparisons that validate these decisions.
