\chapter{Introduction}

\section{Problem Statement}

The beauty and cosmetics industry faces a persistent challenge in providing accurate foundation shade recommendations for diverse skin tones. Traditional methods of shade matching rely heavily on subjective visual assessment by beauty consultants or customers themselves, leading to several critical problems:

\begin{itemize}
    \item \textbf{Inconsistent Matching:} Visual assessment varies significantly based on lighting conditions, individual perception, and experience, resulting in inconsistent and often inaccurate shade recommendations.
    
    \item \textbf{Limited Accessibility:} In-store shade matching requires physical presence and expert assistance, which is not accessible to online shoppers or customers in remote locations.
    
    \item \textbf{Inadequate Representation:} Historical shade ranges have been limited, particularly for deeper skin tones, leading to poor representation and customer dissatisfaction among diverse populations.
    
    \item \textbf{Time and Cost:} The trial-and-error process of finding the right shade is time-consuming and expensive, with customers often purchasing multiple products before finding a suitable match.
    
    \item \textbf{Lack of Standardization:} Different brands use inconsistent naming conventions and classification systems, making it difficult for customers to find equivalent shades across brands.
    
    \item \textbf{Subjective Undertone Detection:} Identifying skin undertones (warm, cool, neutral) is particularly challenging and often leads to mismatches even when the shade depth is correct.
\end{itemize}

These challenges are exacerbated in the digital age, where e-commerce has become the primary shopping channel for many consumers. Without the ability to physically test products, customers rely on product descriptions, reviews, and guesswork, leading to high return rates and customer dissatisfaction.

\vspace{0.5cm}

The problem is further complicated by the lack of inclusive and standardized skin tone classification systems. Traditional systems like the Fitzpatrick scale, developed primarily for dermatological purposes, have only 6 categories and do not adequately represent the full spectrum of human skin tones. This limitation has perpetuated inequities in the beauty industry, with many brands historically offering limited options for deeper skin tones.

\vspace{0.5cm}

Recent research has attempted to address these challenges using artificial intelligence and computer vision techniques. However, existing solutions face their own limitations:

\begin{itemize}
    \item High computational requirements necessitating expensive GPU hardware
    \item Need for large annotated datasets for training deep learning models
    \item Limited accuracy, particularly for deeper skin tones
    \item Lack of production-ready implementations with user-friendly interfaces
    \item Insufficient foundation shade databases covering diverse brands and ranges
\end{itemize}

There is a clear need for an accessible, accurate, and inclusive AI-based solution that can analyze skin tone from photographs and provide personalized foundation shade recommendations without requiring extensive computational resources or training data.

\section{Motivation}

The motivation for developing ShadeAI stems from multiple converging factors in technology, society, and the beauty industry:

\subsection{Technological Advancement}

Recent advances in computer vision, color science, and machine learning have made it feasible to develop accurate skin tone detection systems. The availability of open-source libraries like OpenCV, scikit-learn, and modern web frameworks enables the creation of production-ready applications without requiring extensive resources. The maturity of color difference metrics like CIEDE2000, which closely matches human color perception, provides a scientific foundation for accurate shade matching.

\subsection{Social Equity and Inclusion}

The beauty industry has historically underserved customers with deeper skin tones, leading to a lack of representation and limited product options. The introduction of the Monk Skin Tone (MST) scale by Dr. Ellis Monk provides a more inclusive 10-class classification system that better represents the full spectrum of human skin tones. Developing technology that leverages this inclusive scale contributes to greater equity in the beauty industry and addresses long-standing gaps in product offerings and recommendations.

\subsection{E-commerce Growth}

The rapid growth of online shopping, accelerated by the COVID-19 pandemic, has created an urgent need for virtual beauty consultation tools. Customers shopping online cannot physically test products, leading to uncertainty and high return rates. An AI-based recommendation system can bridge this gap, providing personalized recommendations based on uploaded photographs and improving the online shopping experience.

\subsection{Industry Demand}

Major cosmetic brands are increasingly investing in AI and augmented reality technologies to enhance customer experience. Virtual try-on tools, personalized recommendations, and skin analysis applications are becoming standard features in beauty apps and e-commerce platforms. There is significant industry demand for accurate, scalable, and production-ready skin tone detection systems that can be integrated into existing platforms.

\subsection{Research Gap}

While academic research has explored various approaches to skin tone detection and classification, there is a gap between research prototypes and production-ready systems. Many published solutions require extensive training data, expensive GPU hardware, or lack comprehensive foundation databases. There is an opportunity to develop a practical system that achieves research-grade accuracy using classical computer vision and color science techniques, making it accessible to a broader audience.

\subsection{Personal Experience}

The project team's personal experiences with the challenges of finding suitable foundation shades, particularly for diverse skin tones, provided direct motivation. Witnessing friends and family members struggle with shade matching, waste money on unsuitable products, and experience frustration with limited options highlighted the real-world impact of this problem and the potential value of an effective solution.

\subsection{Academic and Professional Development}

From an educational perspective, this project provides an excellent opportunity to apply knowledge from multiple domains including computer vision, machine learning, color science, software engineering, and user interface design. The project's practical nature and real-world applicability make it an ideal capstone project that demonstrates the ability to solve complex problems using AI and engineering principles.

\section{Project Objectives}

The primary objectives of the ShadeAI project are:

\begin{enumerate}
    \item \textbf{Develop an Accurate Skin Tone Detection System:}
    \begin{itemize}
        \item Implement a robust pipeline for analyzing facial photographs
        \item Achieve color difference (ΔE) below 3.0 for foundation recommendations
        \item Support the inclusive 10-class Monk Skin Tone (MST) scale
        \item Detect skin undertones (warm, cool, neutral) accurately
    \end{itemize}
    
    \item \textbf{Create a Comprehensive Foundation Database:}
    \begin{itemize}
        \item Compile shade information from major cosmetic brands

        \item Provide metadata including undertones and MST ranges
    \end{itemize}

\section{Scope of the Project}

\subsection{In Scope}

The ShadeAI project encompasses the following features and functionalities:

\begin{enumerate}
    \item \textbf{Image Processing Pipeline:}
    \begin{itemize}
        \item Automatic white balance correction using Gray World algorithm
        \item Color space conversions (RGB, CIELAB, HSV, YCbCr)
        \item Image quality assessment and validation
        \item Support for common image formats (JPEG, PNG)
    \end{itemize}
    
    \item \textbf{Face Detection:}
    \begin{itemize}
        \item Haar Cascade classifier for face detection
        \item Fallback mechanisms for challenging images
        \item Face region extraction and validation
    \end{itemize}
    
    \item \textbf{Skin Segmentation:}
    \begin{itemize}
        \item Rule-based segmentation using HSV and YCbCr color spaces
        \item Morphological operations for noise reduction
        \item Skin pixel extraction and validation
    \end{itemize}
    
    \item \textbf{Feature Extraction:}
    \begin{itemize}
        \item Individual Typology Angle (ITA°) calculation
        \item CIELAB color statistics (mean L*, a*, b*)
        \item Color distribution analysis
    \end{itemize}
    
    \item \textbf{Classification:}
    \begin{itemize}
        \item 10-class Monk Skin Tone (MST) classification
        \item ITA°-based threshold classification
        \item Undertone detection (warm, cool, neutral)
        \item Confidence scoring
    \end{itemize}
    
    \item \textbf{Shade Recommendation:}
    \begin{itemize}
        \item CIEDE2000 color difference calculation
        \item Top-5 foundation shade recommendations
        \item Undertone compatibility filtering
        \item Brand and product information
    \end{itemize}
    
    \item \textbf{Foundation Database:}
    \begin{itemize}
        \item 187 foundation shades from 14 brands
        \item CIELAB color values for each shade
        \item Undertone and MST range metadata
        \item JSON format for easy updates
    \end{itemize}
    
    \item \textbf{User Interfaces:}
    \begin{itemize}
        \item Command-line interface with detailed output
        \item Streamlit web application with interactive features
        \item FastAPI REST API with JSON responses
        \item Photo quality checker with real-time feedback
    \end{itemize}
    
    \item \textbf{Documentation:}
    \begin{itemize}
        \item Comprehensive README and usage guides
        \item API documentation with examples
        \item Code comments and docstrings
        \item Accuracy reports and performance analysis
    \end{itemize}
\end{enumerate}

\subsection{Out of Scope}

The following features are not included in the current version but may be considered for future enhancements:

\begin{enumerate}
    \item Deep learning models for segmentation and classification
    \item Real-time webcam capture and analysis
    \item Virtual try-on with augmented reality
    \item Mobile applications (iOS, Android)
    \item Multi-face detection and analysis
    \item Skin condition analysis (acne, wrinkles, etc.)
    \item Personalized skincare recommendations
    \item Integration with e-commerce platforms
    \item User accounts and history tracking
    \item Social media sharing features
\end{enumerate}

\subsection{Limitations}

The system has the following known limitations:

\begin{enumerate}
    \item Requires good quality photographs with proper lighting
    \item Works best with frontal face images
    \item May be affected by heavy makeup or filters
    \item Foundation database limited to 187 shades (though expandable)
    \item Undertone detection uses heuristic methods
    \item No support for video input
    \item Single-face analysis only
\end{enumerate}
